DocumentDB vs. MongoDB: Compatibility, Not Identity

"Little or no change" for most apps — but not every feature exists

What's Compatible

  • MongoDB 4.0, 5.0, and 8.0 API driver compatibility
  • "Vast majority of the applications, drivers, and tools you already use today... can be used with Amazon DocumentDB"
  • DocumentDB 8.0: full wire protocol compatibility with MongoDB 8.0

What's Different

  • "Amazon DocumentDB does not support every MongoDB feature" — built backwards from customer-requested capabilities
  • Functional differences exist between DocumentDB versions themselves (e.g., query planner, projection behavior)
  • Migration path: AWS DMS, or mongodump / mongorestore / mongoimport / mongoexport
Migration Consideration Always check Supported MongoDB APIs, operations, and data types before migrating — assuming 100% parity is the most common migration mistake.Source: Amazon DocumentDB compatibility with MongoDB

DocumentDB vs. DynamoDB: Document Model vs. Key-Value

AWS's own comparison table, read as a two-row decision

Data ModelWhen Would You Use It?What Is It Optimized For?
Document (DocumentDB) Storing JSON-like documents with rich querying abilities across the fields of the documents Semi-structured data as documents with multilayered attributes
Key-value (DynamoDB) Session stores or shopping carts — scale to large data volumes and extremely high request throughput Serverless, NoSQL, fully managed, single-digit millisecond performance at any scale

Rule of Thumb

  • Query across nested document fields, need MongoDB compatibility → DocumentDB
  • Query by known key, need the lowest possible latency at extreme scale → DynamoDB

Security: VPC, TLS, IAM Authentication, Secrets Manager

Four pillars, all documented as first-class AWS features for DocumentDB

VPC Isolation

  • Runs only inside your Amazon VPC — no public internet by default
  • API and interface VPC endpoints (AWS PrivateLink) supported

TLS in Transit

  • "By default, a newly created cluster accepts secure connections only using TLS"
  • DocumentDB 8.0 requires TLS 1.2+ — TLS 1.0/1.1 no longer supported

IAM Authentication

  • Dedicated "Authentication using IAM identity" capability, alongside native DB users
  • Role-Based Access Control (RBAC) for database-level access

Secrets Manager

  • Documented integration: "Password management with Amazon DocumentDB and AWS Secrets Manager"
  • Rotate DB credentials without touching application config

Why Document Model for Catalog/Metadata

  • Semi-structured attributes that vary by item — exactly what document storage is optimized for
  • Rich querying across nested fields, without a fixed relational schema
  • Sits alongside the structured, relational-shaped warehouse layer — not a replacement for it
Enterprise Framing (genericized) In this program's scenario, an aerospace/manufacturing enterprise uses DocumentDB for flexible-schema operational data — product/part catalogs, evolving metadata — alongside its structured data warehouse layer, the same "right tool for the access pattern" principle covered earlier in this program.
